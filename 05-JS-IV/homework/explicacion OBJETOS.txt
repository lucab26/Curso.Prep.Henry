
	多QUE SON LOS OBJETOS?

		UN OBJETO ES UNA COLECCION DE PROPIEDADES, Y UNA PROPIEDAD ES UNA SUMA DE NOMBRE Y VALOR.
		EJ:
			var objeto = {
				prop1: X ,
				prop2: Y,
				...
			}


	多QUE SON LOS METODOS?

		EN LOS OBJETOS, LOS VALORES PUEDEN SER GUARDADOS EN FUNCIONES. LAS FUNCIONES QUE SE GUARDAN DENTRO DE UN OBJETO SE LAS
		CONOCE COMO METODOS.
		EJ:

			var nuevoObjeto = {
				decirHola: function () {
					console.log ("Hola mundo!")
				}
			}

		>nuevoObjeto.decirHola() //// Hola mundo!

	
	多QUE ES EL BUCLE for - in ?

		CUANDO QUEREMOS RECORRER CADA CLAVE Y VALOR DE NUESTRO OBJETO, UTILIZAMOS UNA MATRIZ (BUCLE) Y UNA VARIABLE DE NUMERO INDICE.
		COMO EN LOS OBJETOS NO TENEMOS INDICE NUMERICO, ESTE BUCLE NO FUNCIONARA CON LOS OBJETOS. TENEMOS UN SEGUNDO BUCLE LLAMADO
		"for - in loop". COMIENZA IGUAL, SOLO QUE ENTRE PARENTESIS DECLARAMOS UNA VARIABLE (VARIABLE + in + NOMBRE OBJETO). 
		ESTE BUCLE RECORRERA TODO EL OBJETO HASTA QUE HAYA RECORRIDO CADA CLAVE QUE LO COMPONGA.


	多QUE DIFERENCIA HAY ENTRE LA "NOTACION DE PUNTO (DOT NOTATION)" Y LA "NOTACION DE CORCHETE (BRACKET NOTATION)"?

		- BRACKET NOTATION: NOS PERMITE ACCEDER A LAS PROPIEDADES DEL OBJETO UTILIZANDO "[]".
				    EJ:
					objeto["prop1"], objeto ["prop2"].

		- DOT NOTATION: NOS PERMITE ACCEDER A LAS PROPIEDADES DEL OBJETO UTILIZANDO "." .
				    EJ:
				        objeto.prop1, objeto.prop2.

		LA DIFERENCIA QUE HAY ENTRE ESTAS DOS NOTACIONES, ES QUE LA "DOT NOTATION" ES LITERAL. LO QUE ESCRIBIMOS DEBE ESTAR DENTRO
		DE LAS PROPIEDADES PARA QUE NOS DE UN RESULTADO VALIDO. A DIFERENCIA DE LA "BRACKET NOTATION" QUE  NOS DA MAS LIBERTAD
		A LA HORA DE ASIGNAR VALORES SOBRE LA MARCHA.
